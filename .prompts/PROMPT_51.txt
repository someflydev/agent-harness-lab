# PROMPT_51 - External Project Fixtures And Dry-Run Examples

You are adding external-project fixtures and examples for the portable operator
workflow. Run this in a fresh coding assistant session after Prompt 50 has
been attempted.

## Project Brief

The portable workflow should be provable without requiring a real private
project, network access, provider credentials, or assistant quota. AHL needs
small fixture projects and examples that exercise project status, lifecycle
snippets, context-check behavior, commit-check behavior, and range dry-runs.

This prompt adds fixtures and example docs. It should not make the examples
depend on the user's private gists or notes.

## Fixture Requirements

Fixtures must:

- be clearly fake and safe for public repo use;
- include no secrets, tokens, private gist content, or personal context blobs;
- include prompt files that are structurally valid enough for tests;
- include one project with `AGENT.md`;
- include one project with `CLAUDE.md` but no `AGENT.md`;
- include one project with a prompt gap;
- include a `.context/` example that is small and generic;
- avoid checked-in `.git/` directories.

## Example Requirements

Examples must demonstrate:

- running project status against a fixture;
- generating lifecycle snippets for one prompt;
- using the audit/context-update check;
- keeping commit-plan and make-commits separate;
- running commit-check after commits are made;
- dry-running a prompt range;
- stopping at fresh-session boundaries;
- treating `human-notes.md` as operator-owned if present.

## Test Requirements

Add or update tests so fixture projects exercise:

- project status;
- lifecycle snippets;
- context-check helper if implemented;
- run-range dry-run;
- gapped promptset diagnostics;
- `CLAUDE.md` bootstrap selection.

Keep tests deterministic and offline.

## Constraints

- Do not include real private context blobs or gist URLs.
- Do not check in `.git/` directories.
- Do not run provider CLIs.
- Do not require network access.
- Do not edit `human-notes.md`.
- Do not add dependencies.
- Do not commit unless asked.

## Acceptance Criteria

- The fixtures support reliable offline tests.
- The examples show the portable workflow without overstating automation.
- The fixture docs make clear these are artificial projects.
- All new examples are linked from navigation docs.
