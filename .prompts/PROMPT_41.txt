# PROMPT_41 - Outer Loop Capstone And Operating Baseline

## Project Brief

Phase two should leave AHL with a bounded, inspectable outer-loop capability:
driver contracts, batch plans, dry-runs, gate reports, live sequential runner
MVP, commit planning, resume/failure handling, and a Pi adapter design. The
capstone must audit that implementation honestly and record what is safe to
use.

This prompt should not add a new major feature. It should close small
inconsistencies, run the available local checks, and document the operating
baseline.

## Operating Baseline Requirements

`docs/outer-loop/operating-baseline.md` must define:

- how an operator prepares a clean repo
- how to probe drivers
- how to create a plan for next N prompts
- how to dry-run the plan
- how to run one live prompt step with explicit execution
- how to inspect the gate result
- how to stop and recover
- how to generate a commit plan
- when commit execution is allowed
- what remains manual

It must include a cautious example for Codex:

```bash
python3 scripts/ahl.py outer plan --next 10 --driver codex --model gpt-5.5 --reasoning medium --json
python3 scripts/ahl.py outer dry-run --plan runs/outer-loop/<run-id>/plan.json --json
python3 scripts/ahl.py outer run --plan runs/outer-loop/<run-id>/plan.json --execute --max-prompts 1 --json
```

Do not imply that every local machine has this model, subscription, or CLI
available.
