# PROMPT_48 - One-Prompt Autopilot Runbook And Notes Boundary

You are documenting the human-governed one-prompt autopilot loop for
`agent-harness-lab`. Run this in a fresh coding assistant session after Prompt
47 has been attempted.

## Project Brief

AHL should support the operator's already-proven manual process without taking
over as an autonomous repo builder. The durable workflow is one prompt file at
a time, with explicit audit, optional repair, separated commit planning and
commit execution, post-commit inspection, and a clean fresh-session boundary.

This prompt creates the runbook and examples for that operator loop.

## Runbook Requirements

`runbooks/one-prompt-autopilot.md` must cover:

- discussing a new project in ChatGPT or another planning assistant;
- generating an initial context blob and promptset instructions;
- storing the private initial context blob outside the repo, such as in a
  private gist;
- creating the local project repo and configuring git identity;
- generating initial `.prompts/PROMPT_XX.txt` files and baseline docs;
- preflighting the generated promptset before committing it;
- committing the preflighted promptset;
- running implementation one prompt at a time;
- using lifecycle snippets for run, audit, commit-plan, make-commits, and
  commit-check steps;
- running inline repair when context is still fresh and the issue is bounded;
- creating a handoff only when it is materially useful;
- checking whether `AGENT.md`, `CLAUDE.md`, `.context/`, or `context/` needs
  durable updates;
- preserving a fresh-session reset boundary after each prompt.

## Human Notes Boundary Requirements

`docs/portable-operator/human-notes-boundary.md` must state:

- `human-notes.md` is operator-owned scratch/control-zone content;
- AHL must never automatically edit it;
- AHL must never treat it as authoritative machine state;
- AHL may mention its presence in status output;
- the operator may manually note intentions such as "told AHL to run prompts
  18 through 27";
- reusable snippets may be copied by the operator, but AHL should generate
  snippets directly rather than rewriting notes.
