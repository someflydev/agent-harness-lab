# PROMPT_47 - Commit Message Inspection Helper

You are implementing post-commit inspection for the portable one-prompt
workflow. Run this in a fresh coding assistant session after Prompt 46 has
been attempted.

## Project Brief

The operator keeps "Suggest commits" and "Make the commits" as separate
assistant steps. After commits are made, AHL should help inspect recent commit
messages and grouping evidence without rewriting history automatically.

This prompt adds a local commit-check helper. It should report issues and
guidance only.

## Goal

Add a read-only command that inspects recent commits for prompt-prefix and
message-format hygiene, then reports whether amend or rebase guidance is
needed.

Create or update:

## Command Requirements

Add a command following existing CLI conventions, such as:

```sh
python3 scripts/ahl.py commit check --project <path> --prompt PROMPT_84 --json
python3 scripts/ahl.py commit check --project <path> --prompt PROMPT_84 --range HEAD~10..HEAD --json
python3 scripts/ahl.py commit check --project <path> --last 7 --json
```

The command must inspect commit messages for:

- prompt prefix such as `[PROMPT_84]` in the subject;
- Tim Pope-style short subject, blank line, then body when a body exists;
- subject line length guidance;
- wrapped secondary/body lines;
- no literal `\n` sequences;
- no co-author trailer;
- no accidental generated boilerplate;
- commit count and grouping summary;
- whether changed files in the commit range look plausibly grouped by prompt.

The command may print suggested amend or interactive rebase commands only as
guidance, and only when a clear issue is detected.

## Range Requirements

Support prompt-prefix selection and safe range selectors:

- `--prompt PROMPT_84`, which selects commits with the `[PROMPT_84]` prefix
  without requiring the operator to know how many commits that prompt produced;
- `--last N`, or
- `--range <git-rev-range>`.

Prefer supporting both `--last` and `--range` when it is straightforward. If
`--prompt` is supplied without `--last` or `--range`, inspect prompt-prefixed
commits across the most recent 10 commits by default, and clearly report when
the search may have been truncated. Handle missing git, non-repo directories,
empty history, invalid ranges, and merge commits with clear diagnostics.

## Test Requirements

Add tests using temporary git repositories for:

- passing prompt-prefixed commit with subject/body formatting;
- missing prompt prefix;
- literal `\n` in commit message;
- co-author trailer detection;
- overlong subject or unwrapped body guidance;
- range or last-N selection;
- prompt-prefix selection that finds more than three commits for one prompt;
- no automatic amend, rebase, staging, or commit execution;
- JSON output stable fields.

Skip git-specific tests cleanly if git is unavailable.

## Constraints

- Do not rewrite history.
- Do not run `git commit`, `git commit --amend`, `git rebase`, `git reset`, or
  destructive git commands against the target project or AHL repo.
- Tests may create temporary git repositories and temporary commits solely as
  isolated fixture data for commit-check validation.
- Do not stage files.
- Do not call assistant CLIs.
- Do not require network access or credentials.
- Do not modify `human-notes.md`.
- Do not add external dependencies.

## Acceptance Criteria

- The helper provides a useful post-commit inspection gate.
- It is safe to run from an arbitrary project repo.
- It reports clear issues without taking corrective action.
- The lifecycle snippets include a commit-check step that matches the helper's
  checks.
