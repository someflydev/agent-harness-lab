# PROMPT_19 - Run Records, Fixtures, And Schema Coverage

You are implementing run-record fixtures and schema coverage for
`agent-harness-lab`. Run this in a fresh coding assistant session after Prompts
01 through 18 have been attempted.

## Project Brief

The repo needs concrete machine-readable examples for run records, readiness
reports, promptset indexes, lane records, and traceability records. These
fixtures should make schemas testable and make future automation safer without
introducing a database.

## Startup Instructions

1. Inspect repo state before editing.
2. Read `AGENT.md`, `README.md`, and this prompt.
3. Read `docs/metadata/README.md`, `schemas/`, `registry/README.md`, and
   `templates/runs/run-manifest.md` if present.
4. Run existing script tests and checks if available.
5. Check `git status --short` and preserve unrelated changes.

## Goal

Add representative JSON fixtures and lightweight schema validation support.

## Required Deliverables

Create or update:

- `fixtures/README.md`
- `fixtures/run-records/success.json`
- `fixtures/run-records/blocked.json`
- `fixtures/readiness-reports/ready.json`
- `fixtures/readiness-reports/blocked.json`
- `fixtures/promptset-index/valid.json`
- `fixtures/lane-records/single-lane.json`
- `fixtures/traceability/prompt-to-commit.json`
- `schemas/lane-record.schema.json`
- `schemas/traceability-record.schema.json`
- `scripts/ahl.py`
- `tests/test_ahl.py`
- `docs/metadata/run-record.md`
- `docs/README.md`

## Fixture Requirements

Fixtures must be plausible, compact, and clearly artificial. They should not
claim to represent real completed sessions unless generated by prior prompts.

Include examples for:

- successful prompt run
- blocked prompt run
- ready next-prompt report
- blocked readiness report
- promptset index
- single lane record
- prompt-to-commit traceability record

## Schema Requirements

Add JSON schemas for:

- lane record
- traceability record

Review existing schemas from Prompt 13 and align naming. Do not add complicated
validation rules that are hard to maintain.

## Script Requirements

Enhance `scripts/ahl.py` with a fixture/schema check if not already available.
Because the standard library has no full JSON Schema validator, implement only
lightweight structural checks that are documented honestly:

- JSON parses
- expected top-level fields exist
- fixture type matches expected schema file name
- referenced prompt ids look valid

Do not pretend this is full JSON Schema validation unless you actually
implement it with no dependencies.

## Test Requirements

Update tests to cover fixture checks and at least one invalid fixture case.

## Constraints

- Use only the Python standard library.
- Do not add generated metadata from live git unless Prompt 20 implements it.
- Do not store raw transcripts.
- Do not add external JSON Schema dependencies.
- Do not generate additional prompt files.

## Validation

After editing:

1. Run `python3 -m unittest tests/test_ahl.py`.
2. Run `python3 scripts/ahl.py doctor`.
3. Run the fixture/schema check command.
4. Confirm fixture docs accurately describe lightweight validation limits.
5. Confirm schemas and fixtures use consistent field names.

## Endcap

Before ending the session:

1. Audit fixture and schema deliverables against this prompt.
2. Inspect `.prompts/PROMPT_20.txt` and state whether the repo is ready for it.
3. Perform cheap bridging fixes if traceability collector readiness is blocked
   by missing fixture or schema fields.
4. Create `tmp/HANDOFF.md` only if a real blocker remains.
5. Do not commit unless the operator explicitly asks.
