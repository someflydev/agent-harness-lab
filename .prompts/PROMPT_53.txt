# PROMPT_53 - Portable Operator Capstone And Safety Audit

You are completing the portable operator promptset extension for
`agent-harness-lab`. Run this in a fresh coding assistant session after Prompt
52 has been attempted.

## Project Brief

Prompts 42 through 52 should leave AHL with a practical, local,
human-governed operator workflow for arbitrary project repos that have
`.prompts/`. This capstone should audit the implementation honestly, harden
documentation around safety boundaries, update navigation, and record what is
usable versus still manual.

This prompt should not add a new major feature area.

## Capstone Audit Requirements

`docs/portable-operator/capstone-audit.md` must audit:

- portable invocation and AHL home/project root discovery;
- project status and promptset diagnostics;
- lifecycle snippet generation;
- context-update policy and helper support;
- commit-check helper;
- one-prompt autopilot runbook;
- run-range dry-run orchestration;
- assistant-surface boundaries;
- fixtures and examples;
- rehearsal evidence;
- docs navigation;
- tests and validation;
- known limitations and safety boundaries.

For each area, record:

- status;
- evidence files;
- checks performed;
- gaps;
- whether the gap blocks portable operator baseline use.

## Operating Baseline Requirements

`docs/portable-operator/operating-baseline.md` must explain how an operator:

- installs or invokes AHL from another project repo;
- checks project status;
- generates snippets for one prompt;
- runs the prompt manually in a chosen assistant surface;
- performs audit and next-prompt readiness checks;
- handles inline repair when context is fresh;
- checks context-update candidates;
- requests commit suggestions separately from making commits;
- runs commit-check after commits;
- dry-runs a prompt range;
- stops cleanly at the next prompt boundary.

Include command examples using the actual implemented command names.
The expected portable namespace is `project locate/status`, `lifecycle
snippets/context-check/run-range`, and `commit check`.

## Safety Audit Requirements

The capstone and limitations docs must state:

- every live action that can change files, run assistant commands, or commit
  remains explicit and inspectable;
- no background daemon exists;
- no provider secrets are required;
- no network access is required for portable helpers;
- no expensive APIs are required;
- Claude Code is manual/terminal workflow only, not external subscription API
  automation;
- AHL does not query subscription usage limits programmatically;
- `human-notes.md` is informational and never machine-authoritative;
- commit inspection never rewrites history automatically;
- run-range orchestration is one prompt at a time and dry-run/read-only by
  default.
