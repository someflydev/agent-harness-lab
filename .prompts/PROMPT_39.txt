# PROMPT_39 - Resume, Failure Handling, And Run Ledger Hardening

You are implementing resume and failure-handling support for the phase-two
outer loop. Run this in a fresh coding assistant session after Prompt 38 has
been attempted.

## Project Brief

A sequential runner must fail well. It should be able to stop after a blocked
prompt, explain what happened, preserve enough state to resume safely, and
avoid rerunning successful prompts accidentally. It should also handle ordinary
operator realities: quota exhaustion, context exhaustion, auth failures, local
test failures, dirty worktrees, and interrupted sessions.

This prompt hardens the run ledger and resume commands. It should not expand
the system into a scheduler or daemon.

## Ledger Requirements

`docs/outer-loop/run-ledger.md` must define a ledger with:

- run id
- plan id
- driver id
- model and reasoning/thinking values
- permission posture
- started and ended timestamps
- current status
- per-prompt step records
- command metadata
- validation results
- gate results
- commit plan and commit hashes when present
- resume pointer
- stop reason
- recovery recommendation

The ledger should store derived metadata and concise summaries, not raw
assistant transcripts.

## Failure Classification Requirements

`docs/outer-loop/failure-classification.md` must classify:

- driver executable missing
- driver auth failure
- driver rate limit or quota exhaustion
- driver timeout
- prompt execution failed
- validation failed
- completion audit incomplete
- next-prompt readiness blocked
- unsafe git state
- unexpected changed files
- commit plan refused
- user interruption
- unknown failure

For each class, document:

- whether resume is safe
- whether repair is needed
- what artifact should explain the state
- what the runner must not do automatically

## Script Requirements

Enhance `scripts/ahl.py` with commands such as:

```bash
python3 scripts/ahl.py outer status --run <run-id> --json
python3 scripts/ahl.py outer resume --run <run-id> --dry-run --json
python3 scripts/ahl.py outer recovery-handoff --run <run-id>
```

The commands should:

- read run ledgers from `runs/outer-loop/`
- identify completed, failed, skipped, and pending prompt steps
- refuse resume when the worktree is unsafe
- support dry-run resume planning
- show the next prompt that would run
- avoid rerunning completed prompt steps unless an explicit `--rerun` flag is
  supplied
- create a recovery handoff from template when needed
- degrade clearly when ledger files are missing or malformed

Do not make resume perform live assistant invocation unless it reuses the live
runner command path and still requires explicit execution flags.
