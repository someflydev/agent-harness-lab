# Commands And Routines

This registry makes named commands and docs-only routines discoverable from one
place. A row marked as a helper-script command is executable today through
`scripts/ahl.py`. A row marked as a runbook or routine is currently manual or
assistant-executed.

| Name | Type | Purpose | Inputs | Outputs | Current implementation status | Safety notes |
| --- | --- | --- | --- | --- | --- | --- |
| `doctor` | Helper-script command | Check expected repo foundations and ignore rules. | Repo root, `.gitignore`, foundation paths. | Human summary or JSON checks. | Implemented in `scripts/ahl.py`. | Read-only. |
| `promptset` | Helper-script command | Inspect prompt filenames, numbering, gaps, duplicates, and strict two-digit naming. | `.prompts/PROMPT_*.txt`, optional `--json`. | Human summary or promptset JSON index. | Implemented in `scripts/ahl.py`. | Read-only; does not assess prompt prose quality. |
| `validate` | Helper-script command | Run a lightweight structural validation gate. | Repo foundations, quality directories, promptset data. | Human summary or JSON checks. | Implemented in `scripts/ahl.py`. | Read-only; does not replace prompt-specific validation. |
| `registry list` | Helper-script command | List curated registry JSON files and item counts. | `registry/*.json`, optional `--json`. | Human summary or registry-list JSON. | Implemented in `scripts/ahl.py`. | Read-only; registries are navigation aids. |
| `registry check` | Helper-script command | Validate registry JSON parsing, required item fields, local paths, and prompt ordering. | `registry/*.json`, `.prompts/PROMPT_*.txt`, optional `--json`. | Human summary or registry-check JSON. | Implemented in `scripts/ahl.py`. | Read-only; checked-in files remain source of truth. |
| `resume` | Helper-script command | Print a grounded session context briefing. | Git state, `context/*.md`, `PLAN.md`, `tmp/*.md`. | Human briefing or JSON state. | Implemented in `scripts/ahl.py`. | Read-only; recommendation is advisory. |
| `checkpoint` | Helper-script command | Scaffold missing local context files from examples or minimal built-ins. | `context/*.example.md`, optional `--force`. | `context/TASK.md`, `context/SESSION.md`, `context/MEMORY.md`. | Implemented in `scripts/ahl.py`. | Writes only context files; `--force` overwrites. |
| `scaffold-run` | Helper-script command | Create a timestamped run manifest from the template. | Prompt id, assistant, permission posture, optional output dir. | `runs/<run-id>/run-manifest.md`. | Implemented in `scripts/ahl.py`. | Refuses overwrite unless `--force`. |
| `new-handoff` | Helper-script command | Create a temporary handoff scaffold. | `templates/handoffs/handoff.md`, optional `--force`. | `tmp/HANDOFF.md`. | Implemented in `scripts/ahl.py`. | Use only when bridge decision justifies it; `tmp/` is temporary. |
| `metadata-example` | Helper-script command | Print a skeleton run record for metadata-aware closeout. | Prompt id, assistant, permission posture, prompt batch id, optional `--json`. | Human summary or run-record JSON skeleton. | Implemented in `scripts/ahl.py`. | Skeleton only; repo files and git history remain authoritative. |
| Prompt Execution | Routine | Run one active prompt in a fresh session. | Operator request, `AGENT.md`, active prompt, required docs, git status. | Deliverables, validation evidence, audit and readiness notes. | Defined in `docs/routines/catalog.md` and `runbooks/fresh-session-prompt-run.md`. | One prompt per session unless operator says otherwise. |
| Completion Audit | Routine/runbook | Compare required deliverables against actual artifacts and evidence. | Active prompt, changed files, deliverables, validation notes. | Pass, warning, or blocker statement. | Defined in `docs/routines/catalog.md` and `runbooks/completion-audit.md`. | Do not claim completion for unchecked required deliverables. |
| Next-Prompt Preflight | Routine/runbook | Check whether the immediate next prompt can start cleanly. | Next prompt, current docs, navigation, residual risks. | Ready, risky, or blocked readiness report. | Defined in `docs/routines/catalog.md` and `runbooks/next-prompt-preflight.md`. | Do not implement the future prompt during preflight. |
| Bridge Decision | Routine/runbook | Decide whether a temporary handoff is worth creating. | Completion audit, readiness result, residual issues. | No-handoff decision or narrow handoff need. | Defined in `docs/routines/catalog.md` and `runbooks/bridge-fix-session.md`. | Handoff only for material blockers or warnings. |
| Handoff Generation | Routine/helper-supported | Write temporary continuation context. | Bridge reason, affected files, validation already performed. | `tmp/HANDOFF.md` when needed. | Routine plus `new-handoff` scaffold. | Temporary by default; durable claims need promotion. |
| Repair-Session Startup | Routine/runbook | Start a bounded session for blockers or small gaps. | Defect, affected files, expected fix, stop conditions. | Focused repair plan, changes, validation evidence. | Defined in `docs/routines/catalog.md` and `runbooks/repair-session.md`. | Stop when repair exceeds bounded scope. |
| Promotion Review | Routine/runbook | Decide whether transient knowledge becomes durable memory. | Candidate fact, evidence, target artifact boundary. | Promote, defer, or reject decision. | Defined in `docs/routines/catalog.md` and `runbooks/promotion-review.md`. | Raw assistant chatter is not durable source of truth. |
| Run Closeout | Routine/runbook | Leave repo ready for review or the next fresh session. | Git status, changed files, validation results, residual risks. | Final summary, readiness statement, optional handoff. | Defined in `docs/routines/catalog.md` and `runbooks/run-closeout.md`. | Do not create routine handoffs when final answer is enough. |
| Promptset Health Inspection | Routine/helper-supported | Review prompt sequence health. | Prompt files, docs indexes, completed artifacts. | Findings, warnings, suggested prompt repairs. | Routine plus `promptset` structural check. | Script checks filenames only; prose quality is human-reviewed. |
| Commit Packaging | Runbook | Group completed changes into commits when explicitly asked. | Git status, diff, prompt id, validation evidence. | Commit grouping and commit message summary. | Defined in `runbooks/commit-packaging.md`. | Never commit without operator request. |
| Assumption Diff | Routine | Compare assumptions against durable repo facts. | Assumptions, source docs, repo state. | Confirmed assumptions, conflicts, unknowns. | Defined in `docs/routines/catalog.md`. | Escalate when evidence is unavailable. |
| Scope Boundary Check | Routine | Prevent drift into future prompts or unrelated refactors. | Active prompt, constraints, artifact boundaries. | In-scope, out-of-scope, escalation notes. | Defined in `docs/routines/catalog.md`. | Protect user changes and future-prompt boundaries. |
| Validation Path Discovery | Routine | Identify cheap and relevant checks. | Changed files, README, scripts, package metadata. | Recommended commands and limits. | Defined in `docs/routines/catalog.md`. | Report skipped checks and why. |
