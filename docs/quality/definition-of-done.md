# Definition Of Done

## Done

A prompt is `done` when every required deliverable exists, required content is
present, constraints are honored, validation evidence has been gathered, and
the immediate next prompt has been preflighted.

## Incomplete

A prompt is `incomplete` when required work or evidence is missing but the
remaining work is known and can continue in the same session without changing
scope.

## Blocked

A prompt is `blocked` when completion depends on unavailable prerequisites,
operator input, an unresolved validation failure outside scope, or a conflict
inside the prompt itself.

## Closeout Expectations

Closeout should state the classification, changed files, checks run, skipped
checks, and next-prompt readiness. Do not create a handoff for routine
completion notes that fit in the final answer.
