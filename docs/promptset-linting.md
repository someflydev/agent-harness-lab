# Promptset Linting

`python3 scripts/ahl.py promptset lint` checks prompt files for simple,
inspectable readiness signals. It supports prompt-authoring and prompt-execution
prep by catching structural problems before a fresh assistant session depends
on a prompt.

## Checks

The linter checks:

- filenames match the strict `PROMPT_XX.txt` pattern
- prompt numbers are contiguous
- prompt numbers are not duplicated by malformed aliases such as
  `PROMPT_1.txt`
- each prompt has a `# PROMPT_XX` heading
- each prompt mentions startup instructions
- each prompt mentions required deliverables
- each prompt mentions constraints
- each prompt mentions validation
- each prompt mentions endcap instructions
- each prompt before the final prompt references the immediate next prompt
- `registry/prompts.json` matches the prompt files when that registry exists

The older `python3 scripts/ahl.py promptset` command remains the quick
numbering-only check. Use `python3 scripts/ahl.py promptset lint --json` when a
stable structured result is useful for tests or reports.

## Limits

The linter is intentionally shallow. It does not prove that a prompt is well
written, scoped correctly, semantically complete, or safe to execute. It does
not call a model, rewrite prompts, infer hidden intent, or validate that named
commands are correct for the implementation task.

Failures should be read as structural review items. Passing lint means the
promptset has the expected visible scaffolding; it does not replace the audit
protocol or operator judgment.

## Readiness Scoring

Each prompt receives a transparent readiness score:

```text
present required prompt checks / total required prompt checks
```

The current per-prompt checks are heading, startup instructions, required
deliverables, constraints, validation, endcap, and immediate next-prompt
reference. The final prompt is treated as satisfying the next-prompt-reference
check because there is no later prompt in the set.

Scores are reported with the missing check names so reviewers can see exactly
why a prompt scored below `1.0`. The promptset summary also reports the average
score and total problem count.

## Interpreting Failures

- A malformed filename blocks strict fresh-session ordering and should be
  renamed by the operator or prompt author.
- A gap or duplicate number means the promptset order is ambiguous.
- Missing section signals mean the prompt may rely on unstated context.
- A missing next-prompt reference weakens the endcap preflight loop.
- A registry mismatch means navigation metadata no longer reflects prompt
  files and should be updated or regenerated by the responsible session.

Fix prompt files manually. The linter reports evidence; it does not rewrite the
promptset.

## Prompt-Authoring Use

During prompt-authoring sessions, run the linter before handing a promptset to
fresh execution sessions. It catches common mechanical issues while leaving
scope, language quality, and validation adequacy for human review. For normal
closeout, pair it with:

```sh
python3 scripts/ahl.py promptset lint
python3 scripts/ahl.py validate
python3 -m unittest tests/test_ahl.py
```
